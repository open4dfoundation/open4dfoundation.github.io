About

The Open4D Project

Open4D is developed by the SINRG Lab at Northeastern University as open research infrastructure for time-varying 4D geometry — with the explicit goal of letting research compound instead of resetting with every paper.

Mission

Shared foundations for a fast-moving field

4D geometry — meshes and point clouds whose shape, and even connectivity, changes every frame — is the substrate of volumetric video, telepresence, digital twins, and robot perception. Yet the field's tooling is fragmented: every paper rebuilds IO, metrics, and pipelines from scratch.

Open4D exists to fix that. A stable core provides the data model, IO, and metrics; research modules — codecs, trackers, editors — plug into it and evolve independently; benchmarks make comparisons reproducible; and apps prove the whole stack end-to-end, all the way to real-time playback on an XR headset.

SINRG Lab

The Spatial Intelligence Research Group (SINRG) at Northeastern University works at the intersection of mobile systems, networking, and immersive computing — including volumetric video streaming, XR systems, and machine perception.

Roadmap

Where the platform is heading

Shipped

Core API, IO, and players

The .o4d data model with readers, writers, playback, and conversion tools for meshes and point clouds — pip-installable.

Shipped

Compression modules

N4MC, TSMC, and TVMC integrated; Unity decoder plugin running on Meta Quest 3.

In progress

Benchmarks & baselines

Paper-reproducible evaluation harness with reimplemented baselines and shared metrics.

Planned

C++ core and bindings

Performance-critical implementations with Python bindings for real-time pipelines.

Planned

Streaming stack & end-to-end apps

Adaptive 4D streaming pipelines and reference applications for telepresence and robotics.

Contributing

Get involved

Open4D is an open research platform — contributions are welcome in many forms.

New modules

Codecs, trackers, editors, or any algorithm for time-varying geometry, built on the core API.

Benchmarks & datasets

Reimplemented baselines, new test content, and evaluation configurations.

Metrics

Quality, rate, latency, and temporal-stability measures for systems-aware evaluation.

Documentation

Tutorials, examples, and API documentation that lower the barrier to entry.

Performance

Optimizations, C++ ports, and profiling for real-time and on-device pipelines.

Applications

End-to-end demos — telepresence, robotics, digital twins — that exercise the full stack.

How to cite

If you use Open4D in your research, please cite it.

BibTeX
@software{open4d2026,
  title  = {Open4D: An Open Platform for Time-Varying 4D Geometry Data},
  author = {Spatial Intelligence Research Group},
  year   = {2026},
  url    = {https://github.com/SINRG-Lab/Open4D}
}

A CITATION.cff file with the canonical citation metadata ships in the repository root.
